07. Tool Execution Pipeline

High-Throughput API Interaction & relevance Filtering

On this page

Overview

The **Tool Execution Pipeline** is the system's "Hands." It manages the complexity of interacting with distributed backend APIs while maintaining strict safety invariants. It is designed to handle high-volume read requests in parallel while ensuring that state-modifying write requests are staged for validation.

The Execution Engine

The pipeline uses an asynchronous HTTP client to communicate with the Go Backend. It is "Tool-Agnostic," meaning it derives its behavior from the OpenAPI schema metadata rather than hard-coded functions.

  • Async Core: Leveraging Python's asyncio to handle I/O-bound operations without blocking the graph.
  • Error Mapping: Converts HTTP statuses into semantic "Agent Events" (e.g., 404 becomes a "Resource Not Found" observation for the Planner).

Parallel vs. Sequential Execution

To optimize performance, the pipeline dynamically splits a tool bundle based on the HTTP method.

Tool Type Execution Mode Reasoning
Read (GET) Parallel (gather) Read operations are side-effect free and can be retrieved simultaneously to minimize latency.
Write (POST/PUT) Sequential (staged) Ensures that data flow dependencies (references) are resolved in the correct order.

Relevance Filtering

Industrial API responses can be massive. To prevent "Context Window Bloat," the pipeline applies a Relevance Filter before passing results back to the Planner.

Deterministic Filtering: Automatically strips known metadata fields like created_at, updated_at, or large binary blobs unless explicitly requested.

Semantic Compression: If an output is still too large, it is summarized into key-value pairs that the LLM can easily digest, preserving token count for reasoning.

Idempotency Management

Every tool call is wrapped with a unique idempotency_key. This key is generated based on the session ID, the intent ID, and the tool arguments. This ensures that even if a network retry occurs, the factory backend will never execute the same command twice.
