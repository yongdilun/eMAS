Onboarding Curriculum for New Factory Agent Developers

Overview

Welcome to the Factory Agent team! This path is designed to take you from a newcomer to a confident contributor. Whether you are an AI expert or a backend specialist, this curriculum will bridge the gap between your existing skills and our specialized industrial agent architecture.

Part I: Technology Foundations

Before diving into the codebase, ensure you are comfortable with these core technologies:

Python (Asyncio)

The entire runtime is asynchronous. Understanding await, asyncio.gather, and the event loop is critical.

LangGraph

Our orchestration engine. Learn how Nodes and Edges interact to form a stateful graph.

OpenAPI & REST

We communicate with the Go Backend via dynamically generated tools. Familiarize yourself with RESTful standards.

Part II: Codebase Architecture

The Factory Agent resides in the /factory-agent directory. Here is the layer-by-layer breakdown:

Layer Directory Description
API /api FastAPI routes and SSE event streaming logic.
Graph /graph The StateGraph definition, node functions, and state schema.
Nodes /graph/nodes Individual "worker" nodes: Planner, Guard, Pipeline.
Persistence /persistence MySQL and Redis checkpointer implementations.

Part III: Developer Setup

  1. Environment Config: Copy .env.example to .env and configure your LLM provider and DB credentials.
  2. Install Dependencies: Run pip install -r requirements.txt in a virtual environment.
  3. Launch Infrastructure: Ensure MySQL and Redis are running (use docker-compose if available).
  4. Start the Agent: Run python main.py to launch the FastAPI server and background workers.

Glossary of Terms

Term Definition
AgentState The single source of truth for an active session.
2PC Two-Phase Commit protocol (Stage -> Validate -> Commit).
Intent A decomposed unit of work extracted from a user query.
Decision Guard Non-LLM node that enforces hard safety constraints.
Reference ($ref) A symbolic link used to pass data between tools in a bundle.
