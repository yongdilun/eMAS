08. Human-in-the-Loop (HITL)

The Industrial Safety Brake & Approval Workflow

On this page

Overview

In an autonomous factory, some actions are too critical to be left entirely to AI. The Factory Agent implements a **Human-on-the-Loop** philosophy, where the AI can plan and validate complex operations, but must wait for a "Go/No-Go" signal from a human operator for high-risk mutations.

Native Interrupt System

We leverage LangGraph's Breakpoints to pause graph execution deterministically. When an interrupt is triggered, the entire session state is "frozen" and persisted to the database, allowing the operator to review the plan at their convenience.

The Feature: Native LangGraph interrupts for high-risk write operations.

The Value: This is your "Safety Brake." The agent autonomously reaches a point of "Ready to Commit," generates a complete dry-run report, and then force-pauses until an approval is received.

When is Approval Required?

The system dynamically determines the risk level of a plan. Interrupts are triggered in the following scenarios:

Scenario Trigger Action
Write Operation Any POST/PUT/DELETE in the bundle. Mandatory Interrupt
High-Risk Tool Tool metadata contains is_critical: true. Mandatory Interrupt
Uncertainty Planner confidence score < 80%. Conditional Interrupt

The Approval Lifecycle

1. Interruption

The graph hits the breakpoint before the commit_node. The session status changes to awaiting_approval.

2. Notification

An SSE event (approval_required) is streamed to the frontend, displaying the proposed "Bundle" and the Planner's reasoning.

3. Decision

The operator reviews the dry-run results and clicks Approve or Reject. Approval tokens are injected into the graph's resume payload.
