10. Observability & SSE

Real-Time Telemetry & Transparent Agent Reasoning

On this page

Overview

The Factory Agent provides real-time visibility into its internal reasoning and external actions. This transparency is achieved through a combination of high-frequency **Server-Sent Events (SSE)** for the frontend and deep tracing for developers. The goal is to ensure that operators are never left "in the dark" while the agent plans complex sequences.

SSE Event Streaming

The system uses an asynchronous **Event Bus** (backed by Redis) to stream execution progress. This decoupled architecture allows the graph to run at full speed while the UI receives updates as they happen.

  • Live Reasoning: Every decision made by the Planner is streamed as a "Thought" event.
  • Progress Tracking: Each node transition sends a "Status" event, allowing for a live activity timeline.

Core Event Types

Event Payload UI Impact
step_status_changed node_name, status Updates the color and icon of the active step in the timeline.
agent_thought summary, reasoning Displays a "Thinking..." bubble with the agent's current rationale.
plan_generated List[Action] Renders a preview of the proposed tools for the user to review.
approval_required bundle_id, dry_run_report Triggers the high-risk approval modal.

External Observability

LangSmith Integration

Every session is automatically traced in LangSmith. This provides developers with a full tree-view of the graph execution, including exact token usage, latency per node, and raw LLM prompts/responses.

Audit Database

Unlike transient logs, the Audit DB stores a permanent, tamper-proof record of every Commit action, identifying which operator approved it and the exact timestamp of execution.
